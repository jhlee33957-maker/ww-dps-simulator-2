# 명조 액션 데이터 직접 대조 검수 결과

## 검수 기준

- 프로젝트: `ww-dps-simulator-2(70)`
- 원본: `鸣潮动作数据汇总(5).xlsx`
- 정리본: `명조_종말매트릭스_액션데이터_v6.1_확정수정본(1).zip` 내부 JSON/XLSX
- 원본과 프로젝트에 포함된 원본 XLSX의 SHA-256은 동일하다.
- v6.1의 60 FPS, `派生帧 우선`, 자가 히트스톱 포함, `全局时停`만 차감하는 규칙을 적용했다.
- v6.1의 `lumped_action_level` 정책에 따라 정확한 타격 발생 프레임을 추측하지 않고 액션 총계 1히트로 통합한다. 단, 시뮬레이터의 명시적 hit time은 액션 종료 시점 proxy로 둔다.

- 수정 대상: `data/actions.json` 74개 레코드, `data/transition_actions.json` 4개 레코드
- 고유 액션 ID: 76개
- 액션 ID 및 정책 액션 순서 변경: 없음
- 실제 피해 총계 변경: 3건
  - `aemeath_tune_break`: 13.0 → 12.0
  - `mornye_heavy_geopotential_shift`: 1.4316 → 0.4414
  - `mornye_tune_break`: 16.0 → 12.0
- 나머지 대규모 변경은 액션 시간, 종말 매트릭스 시간, 공명/협주 에너지, 가공된 다단 히트 배열 정리다.
- 모니에의 DEF 계수는 원본 범용 표의 ATK 표기보다 사용자 제공 스킬 설명을 우선하여 유지한다.

## 검증 시 확인된 기존 테스트 영향

- 다단 히트 배열 자체를 고정한 테스트는 `lumped_action_level` 정책에 맞게 총계 검증으로 수정해야 한다.
- 에이메스 Seraphic Duet의 3.0s/2.4167s 옛 타이밍을 고정한 테스트는 185F/145F로 갱신해야 한다.
- 타격 수를 기대하는 계수 테스트는 `sum(hit multipliers) == manifest damage_total` 방식으로 바꿔야 한다.
- 소스 검증 테스트를 삭제하거나 느슨하게 만들면 안 된다. 정확한 매니페스트 값으로 기대값을 교체해야 한다.
